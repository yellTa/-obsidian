# SRP
한 클래스는 하나의 책임만 가져야한다.
- 하나의 책임은 모호하다
	- 클 수 있고, 작을 수 있다.
	- 문맥과 상황에 따라서 다르다.
- 중요한 기준은 변경 -> 변경이 있있을 때 파급 효과가 적으면 단일 책임 원칙을 잘 따른 것

# OCP 개방 폐쇠 원칙
- 소프트웨어 요소는 확장에는 열려있으나, 변경에는 닫혀있어야 한다.
- 다형성을 활용해보자
